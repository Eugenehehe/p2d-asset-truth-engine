P2D gates

01

Physical evidence wins

A verified physical Bin/Rack/U can create a high-confidence update candidate.

02

Spare storage is a location

When the device is found in spare storage, recommend the verified stock Bin and clear Rack/U.

03

Terminal status needs evidence

Official disposed or surplus status plus a receipt supports an update. A missing receipt stays visible as an evidence gap.

04

DNS is a clue, not truth

Active network evidence and a location-shaped hostname can trigger verification, but not an automatic move.

05

Conflicts stop automation

When physical evidence and DNS disagree, the recommendation is routed to manual review.

06

Missing data becomes work

Pending asset tags, DNS, receipts, or physical confirmation become explicit queues with next actions.
